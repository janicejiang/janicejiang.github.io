创建演示应用

设定不要上传的档

(这一步教程里没有, 但我认为这是新建一个项目首先要做的)

  • cp config/database.yml config/database.yml.example
  • 在 .gitignore 文件中加入 config/database.yml
  • 然后init项目

gem 常用格式

  • gem 'sqlite3' 如果没在 gem 指令中指定版本号, Bundler 会自动安装最新版
  • gem 'uglifier', '>= 1.3.0' 安装版本号大于或等于 1.3.0 的最新版 uglifier
  • gem 'pg', '~> 0.18' 只安装最后一个数字变化的版本(例如从 0.18 到 0.21.0)

heroku 常用指令

  • heroku 只能免费创建5个应用, 因为常常用 heroku 做练习, 所以在创建应用之前先用
    heroku apps 确认已创建的应用个数.
  • 如果满额了, 则删除一个 heroku apps:destroy --app rails101
  • 创建应用时指定应用名称 heroku create railstutorial

静态页面

home 动作的路由

get 'static_pages/home' 把发给 static_pages/home 的 GET 请求, 映射到 StaticPages
控制器的 home 动作上.

HTTP 基本操作

(四种请求类型 Rails 都支持)
GET 读取网页
POST 提交表单
PATCH 更新
DELETE 销毁

继承 <

如果是普通的 Ruby 代码, home 方法什么也做不了. 但因为 StaticPagesController 继承自
ApplicationController, 其中的方法在 Rails 框架中就有了特殊意义.
访问 /static_pages/home 时, Rails 会在 StaticPages 控制器中寻找 home 动作, 然后执行该
动作, 因为 home 动作是空的, 所以只会渲染相应的视图.

TDD

如果测试代码特别简短, 倾向于先编写测试
TDD 测试驱动开发流程:

  • 遇红: 先编写一个失败测试
  • 变绿: 然后编写应用代码让测试通过
  • 重构: 最后再根据需要重构代码
