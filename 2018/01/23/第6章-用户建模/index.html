第6章 用户建模

发表于 2018-01-23 | 分类于 railstutorial

models 模型

生成模型

$ rails g model user 模型名是单数
$ rails g controller users 控制器名是复数

id 列

下图中的 id 列在 db/schema.rb 中没有出现过, 这个列是自动生成的, Rails 用这个列作为行的唯一标
识符.

探索数据模型

沙盒模式:
$ rails console --sandbox 在沙盒模式下使用控制台, 退出当前会话后, 对数据库做的所有改动都会回归到原来的状态.

创建user:

  1. user = User.new 只在内存中创建一个对象
  2. user.save 才把对象保存到数据库中, Rails 还会在控制台中显示对应的 SQL 语句.

虽然一般习惯吧创建和保存分成如上所示的两步完成, 不过 Active Record 也允许使用 User.create 方法把这两步合成一步.

查找对象

  • User.find(1) 把用户的 id 传给 User.find 方法，Active Record 会返回 id 为 1 的用户对象; 如果 id 不存在, find 方法会抛出 ActiveRecord::RecordNotFound 异常.
  • User.find_by(email: "mhartl@example.com") Active Record 还支持通过属性查找用户

reload

user.reload.email reload 方法会使用数据库中的数据重新加载对象

validations 验证数据

rails test:models 只运行模型测试

valid? 有效性测试

使用 valid? 方法检查 user 变量的有效性, 如果有一个或多个验证失败, 返回值为 false;
如果所有验证都能通过, 返回 true.
使用 user.errors.full_messages 查看全部错误信息

正则表达式

使用Rubular测试正则表达式, 要把 \A 和 \Z 去掉.

self

其中 self 表示当前用户. 在 User 模型中, 右侧的 self 关键字是可选的, 但左侧的 self 不能省略.
因此可以写成 self.email = email.downcase

安全密码

在模型中调用has_secure_password方法后, 会自动添加如下功能:

  • 在数据库中的 password_digest 列存储安全的密码哈希值;
  • 获得一对虚拟属性, password 和 password_confirmation, 而且创建用户对象时会
    执行存在性验证和匹配性验证; (“虚拟”的意思是模型对象中有属性, 但数据库中没有对应的列)
  • 获得 authenticate 方法, 如果密码正确, 返回对应的用户对象, 否者返回 false.
