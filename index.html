两个小问题

(一) 用户在同一个浏览器中同时打开了两个窗口, 在一个窗口中退出了, 再在另一个窗口退出.

logged_in? 方法调用 current_user 方法判断是否登录, 在第二个窗口退出时,
session[:user_id]/cookies.signed[user_id] 都返回 nil, 代码运行到 current_user 方法的末尾, 返回 nil.
logged_in? 为 false, 则不执行 log_out, 直接跳转到 root_url.

(二) 用户在不同的浏览器中登录, 在一个浏览器中退出, 重启另一个浏览器.

重启时会通过 current_user 方法获取当前用户, 并调用 authenticated? 方法;
由于此时 remember_digest 为 nil, 返回 false, current_user 也返回 nil, logged_in? 自然为 false.
(BCrypt::Password.new(nil) 会抛出异常.)

登录表单

session 会话不是模型, 因此不能创建类似 @user 的变量.
对会话来说, 我们需要指明资源的名称以及相应的 URL.

flash.now

flash.now 专门用于在重新渲染的页面中显示闪现消息; 与 falsh 不同的是, flash.now 中的内容会在下
次请求时消失.

include

在控制器的基类(ApplicationController)中引入辅助方法模块 SessionsHelper, 其中的定义的方法可
以在控制器中使用.

log_in 方法

session[:user_id] = user.id 会在用户的浏览器中创建一个临时 cookie, 内容是加密后的用户 ID.
session 方法创建的是临时会话, 浏览器关闭后立即失效.

current_user 方法

||= 运算符:
@current_user ||= User.find_by(id: session[:user_id])
几乎等效于 @current_user = @current_user || User.find_by(id: session[:user_id])
只有当 @current_user 为假时才会执行后面的表达式; 如果 @current_user 为真, || 会终止执行, 这种方式叫短路计算.

获取固件用户

user = users(:michael) 在测试中获取这个用户

is_logged_in?

is_logged_in? 辅助方法和 logged_in? 功能一样, 测试中不能使用 current_user 方法, 但是可以使用 session 方法. (定义不同的名字以免混淆)

第6章 用户建模

发表于 2018-01-23 | 分类于 railstutorial

models 模型

生成模型

$ rails g model user 模型名是单数
$ rails g controller users 控制器名是复数

id 列

下图中的 id 列在 db/schema.rb 中没有出现过, 这个列是自动生成的, Rails 用这个列作为行的唯一标
识符.

探索数据模型

沙盒模式:
$ rails console --sandbox 在沙盒模式下使用控制台, 退出当前会话后, 对数据库做的所有改动都会回归到原来的状态.

创建user:

  1. user = User.new 只在内存中创建一个对象
  2. user.save 才把对象保存到数据库中, Rails 还会在控制台中显示对应的 SQL 语句.

虽然一般习惯吧创建和保存分成如上所示的两步完成, 不过 Active Record 也允许使用 User.create 方法把这两步合成一步.

查找对象

  • User.find(1) 把用户的 id 传给 User.find 方法，Active Record 会返回 id 为 1 的用户对象; 如果 id 不存在, find 方法会抛出 ActiveRecord::RecordNotFound 异常.
  • User.find_by(email: "mhartl@example.com") Active Record 还支持通过属性查找用户, 如果找不到返回 nil.

reload

user.reload.email reload 方法会使用数据库中的数据重新加载对象

validations 验证数据

rails test:models 只运行模型测试

valid? 有效性测试

使用 valid? 方法检查 user 变量的有效性, 如果有一个或多个验证失败, 返回值为 false;
如果所有验证都能通过, 返回 true.
使用 user.errors.full_messages 查看全部错误信息

正则表达式

使用Rubular测试正则表达式, 要把 \A 和 \Z 去掉.

self

其中 self 表示当前用户. 在 User 模型中, 右侧的 self 关键字是可选的, 但左侧的 self 不能省略.
因此可以写成 self.email = email.downcase

安全密码

在模型中调用has_secure_password方法后, 会自动添加如下功能:

  • 在数据库中的 password_digest 列存储安全的密码哈希值;
  • 获得一对虚拟属性, password 和 password_confirmation, 而且创建用户对象时会
    执行存在性验证和匹配性验证; (“虚拟”的意思是模型对象中有属性, 但数据库中没有对应的列)
  • 获得 authenticate 方法, 如果密码正确, 返回对应的用户对象, 否者返回 false.

Asset Pipeline

(详见 Rails 指南 第 23 章 Asset Pipeline)
app/assets 当前应用的静态资源文件, 因为 custom.scss 只在应用中使用, 所以把它放在
app/assets/stylesheets 目录中.

清单文件
把静态资源文件放在适当的目录中之后, 要通过清单文件 (manifest file) 告诉 Rails 怎么把它们合并成一
个文件, 而且只合并 CSS 和 JavaScript 文件, 不会合并图像.

在应用默认的样式清单文件 application.css 中,
*= require_tree . 会把 app/assets/stylesheets 目录中的所有 CSS 文件都引入应用的 CSS 文件;
*= require_self 会把 application.css 这个文件中的 CSS 也加载进来.

路由

这种形式把发给 /help 的 GET 请求交给 StaticPages 控制器中的 help 动作处理.

因为测试类继承自 ActionView::TestCase, 所以在测试中可以使用 Rails 内置的辅助方法.

自定义 helper

空字符串

由于 app/views/static_pages/home.html.erb 没有 <% provide(:title, "Home") %>
yield(:title) 的返回值为空字符串 "", 并赋值给 page_title
page_title.empty? 返回 true
因此 full_title helper 的返回值为 base_title

参数默认值

因为 def full_title(page_title = '') 中提供了参数的默认值, 即空字符串. 所以,
page_title 参数是可选的, 可以直接调用 full_title 方法不指定参数.

但项目中 yield(:title) 始终有返回值, 因此不指定参数默认值为空也是可以的, 即
def full_title(page_title)

module 模块

模块为我们提供了一种把相关方法组织在一起的方式, 我们可以使用 include 把模块插入其他类中.
编写普通的 Ruby 程序时, 你要自己定义模块, 然后再显示将其引入类中.
但是辅助方法所在的模块会由 Rails 为我们引入, 结果是, full_title 方法自动在所有视图中可用.

block 块

约定:
当块只有一行简单的代码时使用花括号形式
当块是一行很长的代码, 或者有多行时使用 do…end 形式

举个栗子:

从 do 关键字可以看出, 测试的主题其实就是个块, test 方法的参数是一个字符串(测试的描述)和一个
块, 运行测试组件时会执行块中的内容.

散列和符号

(Rails 大量使用这种散列中有散列的形式)

  • 在 Ruby 中, 调用函数可以省略括号
