Asset Pipeline

(详见 Rails 指南 第 23 章 Asset Pipeline)
app/assets 当前应用的静态资源文件, 因为 custom.scss 只在应用中使用, 所以把它放在
app/assets/stylesheets 目录中.

清单文件
把静态资源文件放在适当的目录中之后, 要通过清单文件 (manifest file) 告诉 Rails 怎么把它们合并成一
个文件, 而且只合并 CSS 和 JavaScript 文件, 不会合并图像.

在应用默认的样式清单文件 application.css 中,
*= require_tree . 会把 app/assets/stylesheets 目录中的所有 CSS 文件都引入应用的 CSS 文件;
*= require_self 会把 application.css 这个文件中的 CSS 也加载进来.

测试

rails g integration_test site_layout 生成集成测试模板

测试辅助方法

(参考 第 18 章 Rails 应用测试指南 18.10 测试辅助方法)

针对辅助方法的测试, 只需检测辅助方法的输出和预期值是否一致.
因为测试类继承自 ActionView::TestCase, 所以在测试中可以使用 Rails 内置的辅助方法.
经测试, 即使没有在 ActiveSupport::TestCase 中 include ApplicationHelper, 自定义辅助方法
full_title 也可以调用.

require 'test_helper' 这行代码引入 test_helper.rb 文件, 即加载默认的测试配置. 我们编写的所有
测试都会引入这个文件, 因此这个文件中定义的代码在所有测试中都可用.

自定义 helper

空字符串

由于 app/views/static_pages/home.html.erb 没有 <% provide(:title, "Home") %>
yield(:title) 的返回值为空字符串 "", 并赋值给 page_title
page_title.empty? 返回 true
因此 full_title helper 的返回值为 base_title

参数默认值

因为 def full_title(page_title = '') 中提供了参数的默认值, 即空字符串. 所以,
page_title 参数是可选的, 可以直接调用 full_title 方法不指定参数.

但项目中 yield(:title) 始终有返回值, 因此不指定参数默认值为空也是可以的, 即
def full_title(page_title)

module 模块

模块为我们提供了一种把相关方法组织在一起的方式, 我们可以使用 include 把模块插入其他类中.
编写普通的 Ruby 程序时, 你要自己定义模块, 然后再显示将其引入类中.
但是辅助方法所在的模块会由 Rails 为我们引入, 结果是, full_title 方法自动在所有视图中可用.

block 块

约定:
当块只有一行简单的代码时使用花括号形式
当块是一行很长的代码, 或者有多行时使用 do…end 形式

举个栗子:

从 do 关键字可以看出, 测试的主题其实就是个块, test 方法的参数是一个字符串(测试的描述)和一个
块, 运行测试组件时会执行块中的内容.

创建演示应用

设定不要上传的档

(这一步教程里没有, 但我认为这是新建一个项目首先要做的)

  • cp config/database.yml config/database.yml.example
  • 在 .gitignore 文件中加入 config/database.yml
  • 然后init项目

gem 常用格式

  • gem 'sqlite3' 如果没在 gem 指令中指定版本号, Bundler 会自动安装最新版
  • gem 'uglifier', '>= 1.3.0' 安装版本号大于或等于 1.3.0 的最新版 uglifier
  • gem 'pg', '~> 0.18' 只安装最后一个数字变化的版本(例如从 0.18 到 0.21.0)

heroku 常用指令

  • heroku 只能免费创建5个应用, 因为常常用 heroku 做练习, 所以在创建应用之前先用
    heroku apps 确认已创建的应用个数.
  • 如果满额了, 则删除一个 heroku apps:destroy --app rails101
  • 创建应用时指定应用名称 heroku create railstutorial

home 动作的路由

get 'static_pages/home' 把发给 static_pages/home 的 GET 请求, 映射到 StaticPages
控制器的 home 动作上.

HTTP 基本操作

POST 提交表单
PATCH 更新
DELETE 销毁

继承 <

如果是普通的 Ruby 代码, home 方法什么也做不了. 但因为 StaticPagesController 继承自
ApplicationController, 其中的方法在 Rails 框架中就有了特殊意义.
访问 /static_pages/home 时, Rails 会在 StaticPages 控制器中寻找 home 动作, 然后执行该
动作, 因为 home 动作是空的, 所以只会渲染相应的视图.

  • 变绿: 然后编写应用代码让测试通过
